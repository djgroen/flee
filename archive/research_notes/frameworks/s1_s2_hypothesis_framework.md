# S1/S2 Cognitive Hypothesis Testing Framework\n\n## Current S1/S2 System Analysis\n\nBased on our testing, the current Flee cognitive system has these key features:\n\n### **Cognitive Pressure Formula**\n```\npressure = (conflict_intensity × connectivity) / (recovery_period / 30.0)\n```\n\n### **System Activation Logic**\n- **System 1**: Default mode, fast heuristic decisions\n- **System 2**: Activated when `pressure > conflict_threshold` AND `connections ≥ 3`\n- **Dynamic Switching**: Agents can transition between S1 and S2 based on conditions\n\n### **Behavioral Differences**\n- **S1**: Lower move chances, more randomness, faster decisions\n- **S2**: Higher move chances, more analytical, considers multiple factors\n\n## Testable Hypotheses with Topology Focus\n\n### **H1: Speed vs Optimality Trade-offs**\n\n**H1.1 Multi-Path Decision Quality**\n- **Hypothesis**: S2 agents will choose more optimal routes when multiple paths exist, while S1 agents will choose faster/simpler routes\n- **Topology**: Hub-and-spoke with multiple route options\n- **Test Design**: \n  - Origin → Hub → {Camp_A (close, small), Camp_B (medium, medium), Camp_C (far, large)}\n  - Measure: Route efficiency, destination optimality, decision time\n\n**H1.2 Bottleneck Navigation**\n- **Hypothesis**: S2 agents will anticipate and avoid bottlenecks, S1 agents will get stuck\n- **Topology**: Linear chain with one narrow bottleneck location\n- **Test Design**:\n  - A → B → **Bottleneck** → C → D (camps)\n  - Bottleneck has limited capacity\n  - Measure: Queue formation, alternative route usage\n\n### **H2: Social Connectivity and Information Flow**\n\n**H2.1 Information Cascade in Networks**\n- **Hypothesis**: S2 agents will discover better destinations through social connections, S1 agents rely on local information\n- **Topology**: Tree structure with hidden high-quality camps\n- **Test Design**:\n  - Root → {Branch_A → Hidden_Good_Camp, Branch_B → Obvious_Poor_Camp}\n  - S2 agents should discover Hidden_Good_Camp through connections\n  - Measure: Discovery rates, information propagation speed\n\n**H2.2 Network Density Effects**\n- **Hypothesis**: S2 advantage increases with network connectivity density\n- **Topology**: Grid networks with varying connectivity (sparse vs dense)\n- **Test Design**:\n  - 4x4 grids with different connection densities (25%, 50%, 75%, 100%)\n  - Measure: Movement efficiency vs network density\n\n### **H3: Topology-Dependent Cognitive Advantages**\n\n**H3.1 Linear vs Branching Networks**\n- **Hypothesis**: S1 performs well in linear topologies, S2 advantage emerges in complex branching networks\n- **Topology Comparison**: Linear chain vs Tree vs Grid\n- **Test Design**:\n  - Same population, conflict, and destinations across topologies\n  - Measure: Relative S1 vs S2 performance by topology type\n\n**H3.2 Scale-Free Network Effects**\n- **Hypothesis**: S2 agents will better utilize hub nodes in scale-free networks\n- **Topology**: Scale-free network with high-degree hub nodes\n- **Test Design**:\n  - Generate scale-free topology with clear hub structure\n  - Measure: Hub utilization, path efficiency through hubs\n\n### **H4: Conflict Pattern and Topology Interaction**\n\n**H4.1 Cascading Conflict in Different Topologies**\n- **Hypothesis**: S2 agents will better anticipate conflict spread in well-connected networks\n- **Topology**: Linear vs Star vs Grid topologies\n- **Test Design**:\n  - Same cascading conflict pattern across different topologies\n  - Measure: Evacuation timing, anticipatory movement\n\n**H4.2 Multiple Conflict Sources**\n- **Hypothesis**: S2 agents will better navigate between multiple conflict zones\n- **Topology**: Grid with multiple conflict origins\n- **Test Design**:\n  - 5x5 grid with conflicts starting at corners\n  - Measure: Safe path finding, conflict avoidance\n\n## Synthetic Test Scenarios\n\n### **Scenario 1: The Crossroads Dilemma**\n```\nTopology: Star with 4 camps\n    Camp_A (North): Close, Small capacity, Safe\n    Camp_B (East):  Medium distance, Medium capacity, Moderate safety  \n    Camp_C (South): Far, Large capacity, Very safe\n    Camp_D (West):  Close, Large capacity, Risky (future conflict)\n\nConflict: Gradual escalation at origin\nPrediction: S1 → Camp_A/D (close), S2 → Camp_B/C (optimal)\n```\n\n### **Scenario 2: The Information Cascade**\n```\nTopology: Tree structure\n    Origin → Hub → {Branch_A, Branch_B, Branch_C}\n    Branch_A → Hidden_Excellent_Camp (requires social knowledge)\n    Branch_B → Obvious_Good_Camp (visible to all)\n    Branch_C → Poor_Camp (visible, limited)\n\nSocial Network: S2 agents have connections, S1 agents isolated\nPrediction: S2 discovers Hidden_Excellent_Camp, S1 goes to Obvious_Good_Camp\n```\n\n### **Scenario 3: The Bottleneck Challenge**\n```\nTopology: Linear with bottleneck\n    Origin → Town_A → Bottleneck → Town_B → {Camp_X, Camp_Y}\n    Alternative: Origin → Long_Route → Town_B\n\nBottleneck: Limited capacity, causes queuing\nPrediction: S1 gets stuck at bottleneck, S2 uses alternative route\n```\n\n### **Scenario 4: The Network Density Test**\n```\nTopology: 4x4 Grid with variable connectivity\n    Sparse (25% connections): Few alternative paths\n    Dense (75% connections): Many alternative paths\n\nConflict: Central conflict spreading outward\nPrediction: S2 advantage increases with network density\n```\n\n## Measurable Outcomes\n\n### **Primary Metrics**\n1. **Movement Efficiency**: Distance traveled / optimal distance\n2. **Decision Quality**: Final destination safety/capacity score\n3. **Timing Metrics**: Days to evacuation, days to final settlement\n4. **Route Diversity**: Entropy of path choices\n5. **Bottleneck Avoidance**: Queue time at congested locations\n\n### **Cognitive-Specific Metrics**\n1. **S2 Activation Rate**: Percentage of time in S2 mode\n2. **Transition Frequency**: S1↔S2 switches per agent\n3. **Information Utilization**: Use of non-local information\n4. **Anticipatory Behavior**: Movement before local conflict\n\n### **Topology-Dependent Metrics**\n1. **Hub Utilization**: Usage of high-degree nodes\n2. **Path Length Distribution**: Actual vs shortest paths\n3. **Network Coverage**: Fraction of network explored\n4. **Clustering Effects**: Movement correlation with network structure\n\n## Implementation Strategy\n\n### **Phase 1: Basic Topology Tests**\n1. Implement simple topologies (linear, star, grid)\n2. Test H1 and H3.1 with existing S1/S2 system\n3. Validate that cognitive differences are detectable\n\n### **Phase 2: Social Network Integration**\n1. Enhance social connectivity mechanics\n2. Implement information sharing between connected agents\n3. Test H2 hypotheses\n\n### **Phase 3: Complex Scenarios**\n1. Implement cascading conflicts\n2. Test H4 hypotheses with topology interactions\n3. Generate comprehensive comparison reports\n\n### **Phase 4: Parameter Space Exploration**\n1. Systematic parameter sweeps\n2. Identify phase transitions and critical points\n3. Create dimensionless parameter analysis\n\n## Expected Outcomes\n\n### **Strong Predictions**\n- S2 agents will show higher movement efficiency in complex topologies\n- S1 agents will be faster but less optimal in route selection\n- Social connectivity will amplify S2 advantages\n- Network topology will modulate cognitive mode effectiveness\n\n### **Validation Criteria**\n- Effect sizes > 0.3 (medium effect)\n- Statistical significance p < 0.05\n- Consistent patterns across multiple replications\n- Logical relationship between topology complexity and S2 advantage\n\nThis framework provides a systematic approach to testing dual-process theory in displacement modeling with a strong focus on how network topology influences cognitive decision-making effectiveness.\n"
