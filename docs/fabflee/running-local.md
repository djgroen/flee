# Running simulations with FabFlee

This page covers the main ways to run Flee simulations using FabFlee commands.

---

## Single simulation run

To run a single simulation for a scenario:

```sh
fabsim localhost sflee:<scenario_name>,simulation_period=<days>
```

Example using the built-in Mali 2012 scenario:

```sh
fabsim localhost sflee:mali2012,simulation_period=50
```

For parallel execution (recommended for large simulations):

```sh
fabsim localhost pflee:<scenario_name>,simulation_period=<days>
```

!!! note
    Regular validation runs use `simulation_period=300`. We use 50 days here for speed during testing.

---

## Fetching results

After any run, collect the output files:

```sh
fabsim localhost fetch_results
```

Results are stored in `~/FabSim3/results/`, in a directory typically named `<scenario>_localhost_<id>`.

---

## Plotting output

```sh
fabsim localhost plot_output:<results_dir>,out
```

Example:

```sh
fabsim localhost plot_output:mali2012_localhost_16,out
```

To compare multiple runs of the same conflict scenario:

```sh
fabsim localhost flee_compare:<model1>,<model2>,...,<modelN>
```

---

## Ensemble runs

An ensemble runs multiple simulations simultaneously, each with different inputs. You must first set up a `SWEEP/` directory in your scenario (see [Building scenarios](construction.md)).

```sh
fabsim localhost flee_ensemble:<scenario_name>,simulation_period=<days>
```

For parallel ensemble runs:

```sh
fabsim localhost pflee_ensemble:<scenario_name>,simulation_period=<days>
```

---

## Replicated runs

Flee is non-deterministic. To run multiple identical copies (replicas) of a simulation to account for this variability:

```sh
fabsim localhost flee:<scenario_name>,simulation_period=<days>,replicas=<n>
```

For example, 5 replicas:

```sh
fabsim localhost flee:mali2012,simulation_period=50,replicas=5
```

---

## Coupled simulation (Flee + Flare)

To run Flee coupled with the Flare conflict evolution model:

```sh
fabsim localhost flee_conflict_evolution:<scenario_name>,simulation_period=<days>
```

This runs an ensemble where each variant uses a different conflict trajectory generated by Flare.

---

## DFlee runs

For DFlee (flood-driven displacement) scenarios:

```sh
fabsim localhost pflee:<dflee_scenario_name>,simulation_period=<days>
```

Example:

```sh
fabsim localhost pflee:dflee_test,simulation_period=10
```

---

## Running on a remote machine

Replace `localhost` with your configured remote machine name:

```sh
fabsim eagle_vecma pflee:mali2012,simulation_period=300
```

See [Running on HPC](hpc.md) for setup instructions.
