# Actual S1/S2 Implementation in Flee (Based on Flowchart)\n\n## Corrected Understanding of Flee's S1/S2 System\n\n### **Key Insight: Dynamic Per-Decision Switching**\nAgents don't have a persistent \"S1\" or \"S2\" state. Instead, **each movement decision** triggers a check for System 2 conditions, and the cognitive mode is determined dynamically for that specific decision.\n\n### **Actual Flow (From Flowchart)**\n\n1. **For Each Agent, Each Time Step:**\n   - `calculateMoveChance()` is called\n   - **System 2 Conditions Check:**\n     - Conflict > 0.6?\n     - In recovery period (time >= conflict_start + 10)?\n     - Connections > 3?\n   \n2. **If ALL System 2 conditions are met:**\n   - **System 2 Active** → Pre-calculate provisional route\n   - Return `movechance = 1.0` (always move when S2 active)\n   \n3. **If System 2 conditions NOT met:**\n   - **System 1 Active** → Return to normal `evolve()` processing\n   - Use standard movechance calculation\n\n4. **Route Planning Phase:**\n   - If agent decides to move (`outcome < movechance`):\n     - **If System 2 was active:** Use pre-calculated route\n     - **If System 1 active:** Call `selectRoute()` with `system2_active=False`\n   - `selectRoute()` modifies parameters if `system2_active=True`:\n     - Increases awareness level\n     - Modifies other decision parameters\n     - Restores original parameters after calculation\n\n## Implications for
